First Steps:

Pulling the first images: ⚠

Please enter on commadline: 

docker pull hello-world
 
docker pull hello-world:linux

List the containers:

 
docker image ls

Running the first Hello World Docker container:

 
docker run hello-world:linux

Always remove the containers first, therefore list the containers:

 
docker container ls

Remove the Hello Word Docker containers by there container names:

 
docker container remove ...

Removing the Hello Word Docker images:

 
docker image rm hello-world:latest
 
docker image rm hello-world:linux

Lifecycle of a Docker Container

1. List available images:

Please enter on commadline: 

docker images

2. Pull target image if not already available: 😉

Please enter on commadline: 

docker pull ubuntu/python:3.13-25.04_stable
 
docker image ls

3. Create container from image:

Please enter on commadline: 

docker create --name pythonworld ubuntu/python:3.13-25.04_stable
 
docker container ls -a

4. Start container:

Please enter on commadline: 

docker start pythonworld 
docker ps
 
docker inspect pythonworld

5. Debug container:

Please enter on commadline: 

docker logs -f pythonworld

6. Use container:

Please enter on commadline: 

docker exec -it pythonworld python3

You should now be inside the python3 interpreter: 😉 

python3 --version
 
print("Hello Docker World!")

To exit the python interpreter and the Docker container enter:

 
quit()

To verify that the Docker container is still runnig::

 
docker ps

7. Stop container:

Please enter on commadline: 

docker stop pythonworld

To verify that the Docker container has stoped::

 
docker ps 
docker ps -a

8. Restart container:

Please enter on commadline: 

docker restart pythonworld

To verify that the Docker container is again up and running:::

 
docker ps

9. Pause container:

Please enter on commadline: 

docker pause pythonworld

To verify that the Docker container is pausing:::

 
docker ps

10. Unpause container:

Please enter on commadline: 

docker unpause pythonworld

To verify that the Docker container is again up and running:::

 
docker ps

11. Remove container:

First stop the container before removing: 😉 

docker stop pythonworld

Verify that the container has realy stopped: 😉 

docker ps

Now invoke the remove command: 😉 

docker rm pythonworld

Verify that the container was actually removed: 😉 

docker container ls -a

Linux on the Rocks

,

Pulling the ubuntu image. 🐧

Please enter on commadline: 

docker pull ubuntu:24.04

Checking the image status: 🐧 

docker image ls

Starting a new ubuntu container: 🚀 

docker run -it --name ubuntu-test ubuntu:24.04 bash

Inside the ubuntu container: 😉

 
cat /etc/os-release
 
apt update
 
apt install -y curl
 
curl --version

Exiting the ubuntu container: 😇 

exit

Checking the container status: 🐧

 
docker container ls -a

Restarting the ubuntu container: 🚀 

docker container start -ai ubuntu-test

The good thing is the changes remain in the ubuntu container: 💼 

which curl
 
history

Exiting the ubuntu container for good: 😇 

exit

However, the ubuntu image remains unchanged! ⚠

Only the container has changed, not the image!
To change the image, you have to create a new image based on the changed container.
When you start a new container from the unchanged image, the changes are gone.
This is a key concept of Docker!

Finally let's have a look of a desktop version of ubuntu from within a browser: 😇 

docker run --rm -it --shm-size=512m -p 6901:6901 \ 
						-e VNC_PW=password kasmweb/alpine-320-desktop:1.17.0

Please test in your web browser:

http://localhost:6901

  • Username: kasm_user
  • Password: password

Clean up afterwards: 😉 

docker container rm -f ubuntu-test
 
docker image rm ubuntu:24.04

Debian: More Linux on the Rocks

Pulling the debian image: 🐧

 
docker pull debian:12

Checking the image status: 🐧

 
docker image ls

Checking the container status: 🐧

 
docker container ls -a

In case the container doesn't exist, you can start a new debian container: 🚀

We are mapping port 8888 of the container to port 8888 of the host system.
This is useful in case you want to run a web server or jupyter notebook inside the container.
You can then access it from your host system via localhost:8888
The option -it is used to run the container in interactive mode with a terminal session.
The option --name is used to give the container a name.
The option bash is used to start a bash shell inside the container. 

docker run -it -p 8888:8888 --name debian-test debian:12 bash

In case the container exists you can start the target container right away: 🚀 

docker start -ai debian-test

Inside the debian container: 😉

Let's first check it's a debian image: 🔎 

cat /etc/os-release

Let's ubdate the debian image: 📡 

apt update

Let's install some DNS tools: 💻 

apt install -y dnsutils

Let's install some other esential tools: 😉 

apt install -y curl weget git neovim python3-pip

Let's install some esential development tools: 🔧 

apt install build-essential

Let's install some programming languages: 🔧 

apt install -y golang nodejs npm

Let's install good old java: 🔧 

apt install -y default-jre default-jdk

Let's install Jupyter Notbook for learning python: 🔧 

apt install python3-notebook python3-pandas

Let's create a user to play with Jupyter Notbook for learning python: 🔧 

adduser charly

Exiting the debian container: 😇

 
exit

Checking the container status: 🐧 

docker container ls -a

Restarting the debian container: 🚀 

docker container start -ai debian-test

The good thing is the changes remain in the debian container: 💼 

which jupyter
 
history

Switch to the user to start Jupyter Notbook for learning python: 🔧 

su - charly
 
pwd

Start the Jupyter Notbook for learning python: 🔧 

jupyter notebook

You can now open your web browser and go to http://localhost:8888
You will be asked for a token. You can find the token in the terminal where you started the Jupyter Notebook.
Copy the token and paste it into the web browser.
You can now create a new python notebook and start learning python!

  1. With CTRL-C you can stop Jupyter Notbook
  2. Type exit to leave the user charly
  3. Type exit to leave the debian container

However, the debian image remains unchanged! ⚠

Only the container has changed, not the image!
To change the image, you have to create a new image based on the changed container.
When you start a new container from the unchanged image, the changes are gone.
This is a key concept of Docker!

Finally let's have a look of a desktop version of debian from within a browser: 😇 

docker run --rm -it --shm-size=512m -p 6901:6901 \ 
						-e VNC_PW=password kasmweb/debian-bookworm-desktop:1.17.0

Please test in your web browser:

http://localhost:6901

  • Username: kasm_user
  • Password: password

Clean up afterwards: 😉 

docker container rm -f debian-test
 
docker image rm debian:12
